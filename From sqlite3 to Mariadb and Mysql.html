Migration from Sqlite3 to Mariadb/MySql

  1. Have db file of the sqlite (ex: kuma.db)

  2. Connect to the db with and checks if the database kuma_db exists and retrieves its default character set and collation

    SELECT schema_name, default_character_set_name, default_collation_name

    FROM information_schema.schemata

    WHERE schema_name = 'kuma_db';

    • kuma_db : name of the db

    In my case the sqlite db dont have the supported collation then i change it to utf8mb4_unicode_ci with this cmd:

    ALTER DATABASE kuma_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

  3. Install and setup the tool on the machine that in the same network as the db

    https://github.com/techouse/sqlite3-to-mysql

  4. Use this cmd of the tool for migrations:

    sqlite3mysql \

    -f ./kuma.db \

    -d kuma_db \

    -u kuma_user \

    --mysql-password kuma_password \

    -h <ip_db> -P 3306 \

    --mysql-charset utf8mb4 \

    --mysql-collation utf8mb4_unicode_ci \

    --mysql-insert-method IGNORE \

    -c 1000 \

    -K
